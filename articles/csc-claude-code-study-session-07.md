## 今日のゴール

プラグインという仕組みでClaude Codeの機能をまとめて拡張する方法を理解し、マーケットプレイスからプラグインを1つ導入してみる。

## 座学

### プラグインとは何か

前回学んだAgent Skills(スキル)は、`SKILL.md` というファイル1つで特定の作業を再利用可能にする仕組みでした。プラグインは、このスキルに加えて、カスタムサブエージェント(専門特化した補助人格。次回以降で詳しく扱います)、スラッシュコマンド、MCPサーバーの設定(外部サービスとの接続設定)などを、ひとまとめにして配布できる仕組みです。

言い換えると、スキルが「単機能の道具」だとすれば、プラグインは「道具箱そのもの」です。1つのプラグインを導入するだけで、関連する複数のスキルやコマンドが一度に使えるようになります。

### マーケットプレイスという考え方

プラグインは、GitHubなどのリポジトリを「マーケットプレイス」として登録することで入手します。マーケットプレイスには大きく2種類があります。

- **公式マーケットプレイス**: Anthropicが提供・監修するプラグイン集
- **コミュニティマーケットプレイス**: 個人・企業・オープンソースコミュニティ(OWASPなど)が公開しているプラグイン集

マーケットプレイスを登録すると、そこに含まれるプラグインを `/plugin install` コマンドで検索・インストールできるようになります。マーケットプレイス自体は「プラグインのカタログ」であり、登録しただけでは何も変わりません。カタログの中から実際に使いたいプラグインを選んでインストールする、という2段階の操作になります。

### 導入例で見るプラグインの使い方

ここでは、実際にセキュリティ業務で活用されている導入例を4つ紹介します。いずれも汎用的な位置づけのプラグインで、業種や案件を問わず使えます。

**1. セキュリティレビュー専門のプラグイン集**

OWASPが公開している、AIエージェント自体のセキュリティレビューを目的としたマーケットプレイスがあります。ここから、コードのセキュリティレビュー担当、APIの脆弱性診断担当、依存ライブラリの脆弱性監査担当、モバイルアプリの診断担当といった、特定の診断領域に特化したサブエージェントをまとめて導入できます。1つのプラグインの中に「専門チーム」が同梱されているようなイメージで捉えるとわかりやすいでしょう。診断のたびに手順を毎回説明する必要がなく、「このコードをレビューして」と頼むだけで、専門知識を踏まえた診断が走ります。

**2. 公式のサンプルスキル集**

Anthropicが公開している、スキルの書き方や使い方を学ぶためのサンプル集です。ドキュメント作成支援や資料デザインの支援など、汎用的なスキルが多数含まれています。自分でスキルを作る際のお手本としても参考になります。

**3. 設定点検用のプラグイン**

Claude Code自体の設定(第3回・第4回で扱ったCLAUDE.mdやpermissionsなど)が、公式のベストプラクティスに沿っているかを点検してくれるプラグインです。新しい作業フォルダを使い始めるときに実行すると、設定の抜け漏れに早く気づけます。

**4. 汎用の機能拡張プラグイン**

計画立案やコードレビューの手順を体系化するなど、Claude Codeの標準的な進め方をより丁寧にしてくれる汎用プラグインです。特定の業務に限定されず、日常的な作業の質を底上げする目的で導入されています。

### プラグイン・スキルを導入する際の見極め方

プラグインは、内部的には「あなたの代わりにコマンドを実行したり、ファイルを読み書きしたりする指示書」です。ここは、セキュリティコンサルタントとしての本職の視点がそのまま活きる領域です。導入前に、最低限次のような点を確認する習慣をつけましょう。

- **出所が明確か**: 個人の趣味リポジトリか、組織(OWASPなど)や企業が責任を持って管理しているものか
- **中身が読めるか**: `SKILL.md` やコマンド定義はテキストファイルなので、インストール前に内容を読んで、想定外の外部送信や、権限の過剰な要求がないかを確認できる
- **更新が続いているか**: 長期間更新がなく、依存しているツールが古いままになっていないか

クラスメソッドDevelopersIOのSkills勉強会記事では、良いスキルの目安として「SKILL.md本体は500行以下に収め、詳細な手順は別ファイルに分割する」という基準が紹介されています。これは、Claudeが一度に読み込む分量を抑えて誤読を防ぐという意図もありますが、人間がレビューする際の見通しの良さにもつながります。プラグインを選ぶときも、この「読みやすさ」を1つの判断材料にするとよいでしょう。

自組織で使うプラグインを増やしていく際は、いきなり全員に配るのではなく、まず自分やチームの数名で内容を確認し、業務で問題なく使えることを確かめてから展開する、という段階を踏むことをお勧めします。

## ミニハンズオン

### 手順1: マーケットプレイスを登録する

Claude Code内で以下のように依頼します(コマンドの細かい書式を覚える必要はなく、Claude Codeに操作してもらって構いません)。

```
公式のプラグインマーケットプレイスを登録して、
どんなプラグインが入っているか一覧を見せてください。
```

`/plugin marketplace` のようなコマンドが実行され、登録済みのマーケットプレイス一覧やその中のプラグイン一覧が表示されます。

### 手順2: プラグインを1つインストールする

一覧の中から、業務に関連しそうなプラグインを1つ選び、次のように依頼します。

```
先ほどの一覧にある〇〇というプラグインをインストールしてください。
```

### 手順3: 中身を確認する

インストールしたプラグインが実際に何をするものか、自分の目で確認してみましょう。

```
今インストールしたプラグインに含まれているスキルやコマンドの一覧と、
それぞれの説明を教えてください。
```

この「使う前に中身を確認する」動作を、プラグイン導入時の型として習慣化してください。

## まとめ+チーム資産化アクション(5分)

- プラグインは、スキル・サブエージェント・コマンド・MCP設定をひとまとめにして配布する仕組み
- マーケットプレイスを登録し、その中からプラグインをインストールするという2段階の操作になる
- 導入前に出所・中身・更新状況を確認する習慣は、セキュリティコンサルタントの本職の視点がそのまま活きる

**チームへの持ち帰りアクション**: 今日インストールしたプラグインの内容を簡単にまとめ、チームのナレッジ共有チャンネルに投稿してください。「誰かが良いプラグインを見つけたら、使う前にチームでレビューしてから展開する」という流れを、この機会に定着させましょう。

## ベストプラクティスのひとこと

Anthropic公式のベストプラクティスでは、Claude Codeに与える権限や拡張機能は、信頼できることを確認したうえで少しずつ広げていくことが推奨されています(参考: https://code.claude.com/docs/en/best-practices.md )。プラグインについても同じ考え方が当てはまります。便利そうだからといって手当たり次第に導入するのではなく、内容を確認できる範囲で少しずつ拡張していくことが、結果的に安全かつ長く使い続けられる環境につながります。

## おわりに

社内勉強会は今後も継続していく予定で、この形式で資料を公開していきます。次回はMCP(Model Context Protocol)を使って、Claude Codeを外部のツールやサービスと連携させ、調査からレポート作成までを一気通貫で任せる方法を扱う予定です。プラグインという「拡張の入り口」を押さえたところで、次はいよいよ外部システムとの連携に踏み込んでいきます。
